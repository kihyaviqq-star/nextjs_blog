# Настройка аналитики

## Google Analytics

1. Получите Measurement ID в [Google Analytics](https://analytics.google.com/)
   - Он будет выглядеть как `G-XXXXXXXXXX`

2. Добавьте в `.env.local`:
   ```
   NEXT_PUBLIC_GA_ID=G-XXXXXXXXXX
   ```

## Яндекс.Метрика

1. Получите счетчик в [Яндекс.Метрике](https://metrika.yandex.ru/)
   - ID счетчика будет числом (например: `12345678`)

2. Добавьте в `.env.local`:
   ```
   NEXT_PUBLIC_YANDEX_ID=12345678
   ```

## После настройки

1. Перезапустите dev сервер:
   ```bash
   npm run dev
   ```

2. Проверьте работу:
   - Google Analytics: откройте консоль браузера и проверьте запросы к `google-analytics.com`
   - Яндекс.Метрика: в интерфейсе Метрики проверьте статус счетчика (должен быть зеленым)

## Важно

- Обе системы аналитики загружаются асинхронно (`afterInteractive`), чтобы не замедлять загрузку сайта
- Если переменные окружения не заданы, компоненты просто не будут рендериться
- В production убедитесь, что переменные окружения добавлены в настройки вашего хостинга (Vercel, etc.)
